================================================================
     ENVIRONMENT FILES - IMPORTANT NOTE
================================================================

THE .env.example FILES ARE HANDLED AUTOMATICALLY!
--------------------------------------------------

All installation scripts (setup.sh, setup.bat, install-vps.sh)
include built-in templates that create .env files automatically.

You don't need to:
- ❌ Copy .env.example manually
- ❌ Worry about missing .env.example files
- ❌ Create .env files yourself

The scripts do it all automatically! ✅


HOW IT WORKS:
-------------
1. Script checks if balkly-api/.env exists
2. If .env.example exists → copies it
3. If .env.example missing → creates from built-in template
4. Either way, you get a working .env file!


AFTER INSTALLATION:
-------------------
You just need to ADD YOUR API KEYS:

Edit: balkly-api/.env
Add:
  STRIPE_KEY=pk_test_...
  STRIPE_SECRET=sk_test_...
  MAIL_PASSWORD=YOUR_SENDGRID_KEY
  OPENAI_API_KEY=sk-...


ON UBUNTU VPS:
--------------
nano /var/www/balkly/balkly-api/.env

Add your keys, save, then:
docker-compose restart
